Массив

Массив – непрерывный кусок памяти, хранящий в себе фиксированное число элементов одинакового типа. Каждый элемент массива имеет индекс – его порядковый номер. Индексация начинается с 000.

Для всех элементов выделяется одинаковое целое число байт, определяемое по типу.

Для получения элемента типа T с индексом iii в массиве a нужно прочитать (size of T) байт начиная с позиции a + i * (size of T). При этом a является указателем на начало массива.

Вот пример хранения массива в памяти. Указатель на начало массива a = 0x8E53, в массиве 666 элементов. Если тип элементов short (каждый 222 байта), то a = [583, 250, 18, 14146, -40, 0].

Получение значения из ячейки памяти и запись значения в ячейку по адресу выполняются за время O(1)O(1)O(1). Поэтому доступ и изменение произвольных элементов выполняется за время O(1)O(1)O(1).

Статический массив

Статический массив – массив, у которого привязанный к нему кусок памяти неизменен. Таким образом, размер массива тоже остается неизменным.

Статические массивы должны знать свой размер на момент создания. Если массив объявлен статически (например, в глобальной области видимости или как статический атрибут), то он размещается в статической памяти. Локально объявленные массивы размещаются на стеке.

Дополнительное пространство

Можно заранее выделить некоторое количество памяти и постепенно ее заполнять. Таким образом длина может быть переменной, но она ни в коем случае не может превышать вместимость (то, сколько мы изначально выделили). Удаление с конца и вставка в конец здесь будут работать за O(1)O(1)O(1).

Статический массив с возможностью ограниченного расширения реализовывается так

type static_array: T* head int size int capacity constructor(self, capacity): self.size = 0 self.capacity = capacity self.head = memory_allocate(capacity, size of T) destructor(self): memory_free(self.head) method append(self, item): assert self.size < self.capacity *(head + self.size) = item self.size++ method delete(self): assert self.size > 0 self.size-- operator self[index]: return *(self.head + index) operator self[index] = value: *(self.head + index) = value

Динамический массив

Динамический массив – массив, который может изменять свой размер в зависимости от количества элементов, которые в нем находятся.

Кроме операции доступа и изменения произвольного элемента есть еще добавление элемента в конец и удаление элемента из конца, которые работают за амортизированное время O(1)O(1)O(1).

В динамическом массиве 3 поля: указатель на первый элемент *head (или статический массив), размер динамического массива size и его максимальная вместимость capacity.

type dynamic_array: T* head int size int capacity constructor(self, int capacity): self.size = 0 self.capacity = capacity self.head = memory_allocate(capacity) destructor(self): memory_free(self.head)

Динамический массив умеет изменять свой размер: расширять и уменьшать. Для изменение размера нужно выделить новый участок памяти нужной длины, скопировать нужные значения в новый участок, перезаписать указатель head и освободить предыдущую занимаемую память.

method resize(self, int capacity): self.head = memory_reallocate(self.head, capacity) self.capacity = capacity

Вставлять элемент в конец динамического массива нужно так же, как в статический: увеличить размер и задать значение последнему элементу. Когда внутренний статический массив оказывается полностью заполненным (когда size == capacity), его необходимо расширить.

Для достижения амортизированного времени вставки O(1)O(1)O(1) (с учетом расширения), увеличивать вместимость массива необходимо в какое-то константное количество раз.

Оптимальная константа увеличения равна 222 (баланс между временем и памятью). Хотя, встречаются и другие реализации – в Java, например, эта константа равна 1.51.51.5.

method append(self, item): if self.size == self.capacity: self.resize(self.capacity * 2) self.head[self.size] = item self.size++

При удалении последнего элемента мы просто уменьшаем размер динамического массива на 111.

Чтобы сэкономить память, можно уменьшать вместимость массива в эту же константу раз. Однако, для достижения амортизированного времени O(1)O(1)O(1) сразу обоих операций вставки и удаления, условие на уменьшение вместимости должно быть более сильным.

Например, вместимость массива можно уменьшить в 222 раза, если его размер в 444 раза меньше вместимости. Таким образом, если вставить, удалить, вставить, удалить элемент, массив изменит свою вместимость лишь один раз.

method delete(self): assert self.size > 0 if size * 4 < capacity: self.resize(capacity / 2); self.size--

Базовые операции

Вставка. Если мы вставляем элемент не в конец массива, а в произвольное место, то остальные элементы придется сдвигать. Временная сложность этой операции O(n)O(n)O(n).

function insert(array a, int index, value): assert a.size < a.capacity for i = a.size, i > index, i--: a[i] = a[i-1] a[index] = value a.size++

Удаление. Удаление произвольного элемента, аналогично вставке, требует сдвига нескольких элементов. Временная сложность тоже O(n)O(n)O(n).

function remove(array a, int index): assert a.size > 0 for i = index, i < a.size: a[i] = a[i+1] a.size--

Поиск. Для поиска значения в массиве придется пройтись по всему массиву. Сложность O(n)O(n)O(n).

function find(array a, value): for x with index i in a: if x == value: return i
