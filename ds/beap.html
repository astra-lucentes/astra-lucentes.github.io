Двуродительская куча

А если в куче нужен поиск?

Двуродительская куча (bi-parental heap, beap) – полное двоичное псевдодерево, в котором у каждого узла до двух детей и до двух родителей. Значение любого узла не больше значений его потомков.

Двуродительская куча позволяет извлекать минимум, вставлять, удалять и искать элементы за O(n)O\b(\sqrt n\b)O(n​). Также поддерживается нахождение минимума за O(1)O(1)O(1) и максимума за Θ(n)\Theta\b(\sqrt n\b)Θ(n​).

Хранить двуродительскую кучу можно в массиве без дополнительной информации.

Все уровни хранятся по порядку, от верхнего к нижнему. Связи определяются по индексам внутри уровней. Элементы на уровне hhh хранятся по индексам с h(h−1)2\frac{h(h-1)}{2}2h(h−1)​ по h(h+1)2−1\frac{h(h+1)}{2} - 12h(h+1)​−1, уровни нумеруются с 111.

Удобнее работать с положением элемента не по его индексу в общем массиве, а по паре чисел (h,k)(h, k)(h,k), где hhh – номер уровня, а kkk – индекс в этом уровне. Индекс в общем массиве вычисляется как h(h−1)2+k\frac{h(h-1)}{2}+k2h(h−1)​+k.

У элемента (h,k)(h, k)(h,k) родителями являются (h−1, k)(h-1,\, k)(h−1,k) и (h−1, k−1)(h-1,\, k-1)(h−1,k−1) (если индекс выходит за границы уровня, то соответствующего родителя у элемента нет), а детьми являются (h+1, k)(h+1,\, k)(h+1,k) и (h+1, k+1)(h+1,\, k+1)(h+1,k+1).

Операции над кучей

Восстановление свойств

Если в куче изменяется один элемент, то она может перестать удовлетворять свойству упорядоченности.

Для восстановления этого свойства нужны две процедуры shift_up и shift_down (просеивание).

Алгоритм просеивания вниз

  1. Если просеиваемый элемент больше, чем его дети, то меняем его с наименьшим из его детей.

  2. Продолжаем просеивать этого ребенка, если еще не дошли до конца кучи.

private method shift_down(array b, int h, int k): while h < height of b: l = (h + 1) * h / 2 + k r = (h + 1) * h / 2 + k + 1 i = h * (h - 1) / 2 + k if r < size of b and b[r] < b[l]: j = r h, k = h + 1, k + 1 else: j = l h, k = h + 1, k if b[i] <= b[j]: break swap b[i], b[j]

Алгоритм просеивания вверх

  1. Если просеиваемый элемент меньше его родителей, то меняем его с наибольшим из его родителей.

  2. Продолжаем просеивать этого родителя, если еще не дошли до верха кучи.

private method shift_up(array b, int h, int k): if height of b: return while h > 1: l = (h - 1) * (h - 2) / 2 + k - 1 r = (h - 1) * (h - 2) / 2 + k i = h * (h - 1) / 2 + k if k == 0: j = r h, k = h - 1, k else if k == h - 1: j = l h, k = h - 1, k - 1 else if b[r] > b[l]: j = r h, k = h - 1, k else: j = l h, k = h - 1, k - 1 if b[i] <= b[j]: break swap b[i], b[j]

Обе процедуры выполняются за время O(высота кучи)=O(n)O(\text{высота кучи}) = O\b(\sqrt n\b)O(высота кучи)=O(n​).

Извлечение минимального элемента

Алгоритм извлечения минимального элемента

  1. Сохраняем значение корневого элемента для дальнейшего возврата

  2. Помещаем последний элемент в корень

  3. Выполняем shift_down для корня

  4. Возвращаем сохраненное значение

method extract_min(array b): maximum = b[0] b[0] = b[-1] delete b[-1] b.shift_down(1, 0) return maximum

Операция выполняется за время O(n)O\b(\sqrt n\b)O(n​), равное времени выполнения shift_down.

Добавление элемента

Алгоритм добавления элемента

  1. Добавляем элемент в конец кучи

  2. Выполняем shift_up для этого элемента

method push(array b, value): b.append(value) b.shift_up(height of b, size of b - 1)

Операция выполняется за время O(n)O\b(\sqrt n\b)O(n​), равное времени выполнения shift_up.
