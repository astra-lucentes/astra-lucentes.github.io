Бинарное дерево поиска

Бинарное дерево поиска (BST, binary search tree) – бинарное дерево, в котором у каждого узла все его левые дети его меньше, а все правые дети его больше.

BST позволяет добавлять, изменять, удалять и искать элементы за время O(высота дерева)O(\text{высота дерева})O(высота дерева).

Его можно построить на любом отношении частичного порядка. Это отношение порядка может быть задано не на самих данных, а на их ключах.

Хранение

BST состоит из узлов (вершин), имеющих следующую структуру:

struct node: T value node left node right

Операции над деревом

Поиск

В BST можно очень просто и достаточно быстро, за время O(высота дерева)O(\text{высота дерева})O(высота дерева), искать элементы.

Мы постоянно пользуемся упорядоченностью BST. Если искомый элемент меньше, чем корневой элемент, то мы продолжаем поиск в левом поддереве, если больше, то продолжаем поиск в правом поддереве, а если равен, то мы нашли элемент.

function search(node root, value): if root == none or root.value == value: return root if value < root.value: return search(root.left, value) else: return search(root.right, value)

Рекурсию можно развернуть и написать более эффективный, итеративный поиск

function search(node root, value): while root != none and root.value != value: if value < root.value: root = root.left else: root = root.right return root

Поиск минимума и максимума

Поскольку элементы левого поддерева меньше корня, самый левый узел дерева будет иметь минимальное значение. Аналогично, самый правый узел дерева имеет максимальное значение. Для поиска минимума и максимума достаточно постоянно двигаться влево или вправо.

Время выполнения обоих операций O(высота дерева)O(\text{высота дерева})O(высота дерева).

function min(node root): while root.left != none: root = root.left return root
function max(node root): while root.right != none: root = root.right return root

Обходы дерева

Обход по порядку

Необходимо вывести все узлы BST в отсортированном порядке. Для этого, по свойству BST, необходимо обойти в отсортированном порядке левое от корня поддерево, затем пройти по корню, затем обойти в отсортированном порядке правое от корня поддерево. Временная сложность Θ(n)\Theta(n)Θ(n).

iterator inorder(node root): if root == none: stop yield from inorder(root.left) yield root yield from inorder(root.right)

Прямой обход

Необходимо вывести все узлы BST в порядке: вершина, все левое поддерево, все правое поддерево. Временная сложность Θ(n)\Theta(n)Θ(n).

iterator preorder(node root): if root == none: stop yield root yield from preorder(root.left) yield from preorder(root.right)

Обратный

Необходимо вывести все узлы BST в порядке: все левое поддерево, все правое поддерево, вершина. Временная сложность Θ(n)\Theta(n)Θ(n).

Такой обход полезен, например при удалении всех вершин, ведь сначала обходятся вершины без детей, а дети всех последующих вершин гарантированно были выведены ранее.

iterator postorder(node root): if root == none: stop yield from preorder(root.left) yield from preorder(root.right) yield root
