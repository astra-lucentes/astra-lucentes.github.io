Построение

Построение через изменение

Можно построить в лоб по определению за O(n2)O(n^2)O(n2), а можно выполнить nnn раз функцию modify.

В начале ttt состоит из нулей. Для каждого элемента массива aaa мы выполняем операцию modify, тем самым строим дерево за O(nlog⁡n)O(n \log n)O(nlogn).

function build(array a[n]) -> array: array t[n] = [0, ..., 0] for i = 0; i < n: while i < n: t[i] += a[i] i = i | (i + 1) return t

Линейное построение

Можно ускорить построение дерева Фенвика, сделав его линейным.

Каждый элемент дерева Фенвика представляет собой сумму на непрерывном отрезке исходного массива. Значит на моменте инициализации можно сначала подсчитать префиксные суммы, а затем по ним составить дерево Фенвика.

Временные затраты O(n)O(n)O(n) на создание кумулятивного массива и n⋅O(1)n \cdot O(1)n⋅O(1) на создание дерева Фенвика. Итого время O(n)O(n)O(n) и память O(n)O(n)O(n) на кумулятивный массив.

function build(array a[n]) -> array: array t[n] = [0, ..., 0] array s[n] = [0, ..., 0] s[0] = a[0] for i = 1; i < n: s[i] = s[i-1] + a[i] for i = 0; i < n: j = i & (i + 1) - 1 if j >= 0: t[i] = s[i] - s[j] else: t[i] = s[i] return t

Лишних затрат памяти можно избежать. Для инициализации tit_iti​ нам потребуются только элементы префиксных сумм sjs_jsj​, где j⩽ij \le ij⩽i. Значит, мы можем перестроить кумулятивный массив в дерево Фенвика, начиная с конца.

function build(array a[n]) -> array: array t[n] = [0, ..., 0] t[0] = a[0] for i = 1; i < n: t[i] = t[i-1] + a[i] for i = n - 1; i > 0; i--: j = i & (i + 1) - 1 if j >= 0: t[i] -= t[j] return t
