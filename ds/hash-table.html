Хеш-таблица

Хеш-таблица – ассоциативный массив, хранящий данные в обычном массиве. В хеш-таблице для вычисления индекса используется хеш-функция от ключа. Позволяет за ожидаемое время O(1)O(1)O(1) добавлять, удалять и искать элементы.

Для достижения эффективного времени выполнения всех операций хеш-функция должна иметь очень маленькое количество коллизий. Однако, эти коллизии неизбежны, поэтому придется их как-то разрешать, чтобы не потерять и не исказить данные.

Разрешение коллизий

Связные списки

Самый простой способ разрешения коллизий - хранить в массиве связные списки. В связном списке, находящемся по индексу iii будут хранятся только объекты с хешем iii.

Для добавления объекта по ключу нужно сначала вычислить хеш ключа, тем самым получить индекс. Затем просто вставить объект в конец связного списка, хранящегося по полученному индексу.

При поиске объекта по ключу вычисляется хеш и получается индекс, затем объект ищется в соответствующем связном списке. Удаление аналогично.

Если максимальное количество элементов в каждом связном списке mmm, то удаление и поиск занимают время O(m)O(m)O(m), в вставка всегда занимает время O(1)O(1)O(1).

Деревья поиска

Вместо связных списков из предыдущего подхода можно использовать сбалансированные деревья поиска. Время поиска и удаления уменьшается до O(log⁡m)O(\log m)O(logm).

Линейное разрешение

В этом подходе объекты будут храниться непосредственно в массиве.

При попытке разместить элемент в ячейку iii может оказаться, что эта ячейка уже занята. Тогда можно просматривать следующие ячейки i+1i+1i+1, i+2i+2i+2 и так далее, пока не найдется пустая ячейка.
