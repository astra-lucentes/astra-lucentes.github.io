Двоичная куча

Двоичная куча или пирамида – полное двоичное дерево, в котором значение любой вершины не меньше (или не больше), чем значения её потомков.

В кучах можно использовать любое отношение частичного порядка ⋄\diamond⋄, тогда свойство кучи будет таким:

Для любой вершины v и каждого её потомка x верно v⋄x.\text{Для любой вершины}~ v ~\text{и каждого её потомка}~ x ~\text{верно}~ v \diamond x \text{.}Для любой вершины v и каждого её потомка x верно v⋄x.

Куча позволяет изменять, добавлять элементы, а так же извлекать максимум (а в общем случае элемент, для которого верно бинарное отношение со всеми другими элементами) за O(log⁡n)O(\log n)O(logn).

Хранить элементы двоичной кучи можно в массиве без дополнительной информации.

Все слои дерева хранятся по порядку, от верхнего к нижнему. При этом у элемента с индексом iii дети имеют индексы 2i+12i + 12i+1 и 2i+22i + 22i+2. Максимальный элемент имеет индекс 000. Высота кучи ⌈log⁡2(n+1)⌉\bl\lceil \log_2 (n+1) \br\rceil⌈log2​(n+1)⌉.

Операции над кучей

Восстановление свойств

Если в куче изменяется один элемент, то она может перестать удовлетворять свойству упорядоченности.

Для восстановления этого свойства нужны две процедуры shift_up и shift_down (просеивание).

Алгоритм просеивания вниз

  1. Если просеиваемый элемент меньше, чем его дети, то меняем его с наибольшим из его детей.

  2. Продолжаем просеивать этого ребенка, если еще не дошли до конца кучи.

private method shift_down(array h, int i): while 2 * i + 1 < size of h: left = 2 * i + 1 right = 2 * i + 2 if right < size of h and h[right] > h[left]: j = right else: j = left if h[i] >= h[j]: break swap h[i], h[j] i = j

Алгоритм просеивания вверх

  1. Если просеиваемый элемент меньше, чем его родитель, то меняем этот элемент с его родителем.

  2. Продолжаем просеивать родителя, если еще не дошли до верха кучи.

private method shift_up(array h, int i): while h[i] > h[(i - 1) / 2]: swap h[i], h[(i - 1) / 2] i = (i - 1) / 2

Обе процедуры выполняются за время O(высота кучи)=O(log⁡n)O(\text{высота кучи}) = O(\log n)O(высота кучи)=O(logn).

Извлечение максимального элемента

Алгоритм извлечения максимального элемента

  1. Сохраняем значение корневого элемента для дальнейшего возврата

  2. Помещаем последний элемент в корень

  3. Выполняем shift_down для корня

  4. Возвращаем сохраненное значение

method extract_max(array h): maximum = h[0] h[0] = h[-1] delete h[-1] shift_down(h, 0) return maximum

Операция выполняется за время O(log⁡n)O(\log n)O(logn), равное времени выполнения shift_down.

Добавление элемента

Алгоритм добавления элемента

  1. Добавляем элемент в конец кучи

  2. Выполняем shift_up для этого элемента

method push(array h, value): h.append(value) shift_up(h, size of h - 1)

Операция выполняется за время O(log⁡n)O(\log n)O(logn), равное времени выполнения shift_up.

Удаление произвольного элемента

Обычная куча не поддерживает удаление произвольного элемента по его значению.

Эту операцию можно реализовать с помощью хеш-таблицы, в которой для каждого элемента поддерживается ссылка на его место. Не самый удобный вариант. Существует более элегантный и простой метод с использованием только куч.

Пусть у нас есть куча h, для которой мы хотим реализовать произвольное удаление элемента. Заведем еще одну кучу d, в которой будем хранить удаляемые элементы.

Когда пришел запрос на удаление xxx, с кучей h мы ничего не делаем, а просто добавляем xxx в кучу d.

При обращении к максимальному элементу кучи h может оказаться так, что он уже был удален. Тогда пока максимальные элементы куч h и d совпадают, извлекаем их, а затем продолжаем работу в штатном режиме.
