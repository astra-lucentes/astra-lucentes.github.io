Связный список

Связный список – структура данных, состоящая из узлов, содержащих помимо данных ссылку на следующий и/или предыдущий узел.

Связный список позволяет вставлять и удалять узлы из произвольного места за O(1)O(1)O(1).

struct node: data node next

Узлы связного списка можно обернуть в дополнительный тип linkedlist, который имеет только один атрибут - ссылку на первый элемент списка.

type linkedlist: node head constructor(self): self.head = none

Базовые операции

Вставка. Для того, чтобы вставить узел после какого-то узла, надо перезаписать ссылки.

function insert_after(node current, node what): what.next = current.next current.next = what

Удаление. Чтобы удалить узел, следующий за текущим, нужно перезаписать next у текущего и физически удалить нужный узел.

function delete_after(node current): to_delete = current.next current.next = current.next.next delete to_delete

Обход. Произвольного доступа в этой структуре данных нет, а обход реализуется циклом. Переменная head здесь побывает каждым узлом по порядку.

function walk(node head): while head != none: (do something) head = head.next

Поиск цикла

Связный список может оказаться цикличным, когда next у последнего узла ссылается на один из существующих узлов.

Для того, чтобы понять, является ли связный список цикличным, можно воспользоваться алгоритмом Флойда поиска цикла, рассматривая связный цикл как функциональный граф.

function has_cycle(node head): slow = head fast = head.next while slow != fast: if fast == none or fast.next == none: return false slow = slow.next fast = fast.next.next return true

Если связный список не имеет цикла, то fast дойдет до конца первым и мы вернем false. Если же цикл есть, то расстояние между fast и slow будет сокращаться на 111 каждую итерацию, и они когда-нибудь встретятся.

С помощью этого же алгоритма можно найти начало цикла, если запомнить meet – момент встречи быстрого и медленного указателя.

function cycle_beginning(node head, node meet): first = head.next second = meet.next length = 1 while first != second: first = first.next second = second.next length++ return first, length

Связный список на индексах

Вместо хранения ссылок можно хранить индексы одного массива. Все узлы являются элементами этого массива, а для получения узла достаточно обратиться по соответствующему индексу.

Добавляются элементы в конец за O(1)O(1)O(1), доступ по индексу осуществляется за O(1)O(1)O(1)

record node: int next int prev data

Такой способ немного экономит память (на индексы выделяется меньше байт, чем на ссылки), весь список можно легко перемещать по памяти и сериализовать.
