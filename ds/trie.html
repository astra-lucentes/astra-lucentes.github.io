Бор

Бор – ассоциативный массив, ключи которого являются строками над конечным алфавитом Σ\SigmaΣ мощности σ\sigmaσ. Может также хранить только строки без значений.

Бор представляет собой корневое дерево, каждое ребро которого помечено каким-то символом, причем ребра, выходящие из одной вершины, содержат разные символы. Некоторые узлы отмечены (являются терминальными). Бор содержит данную строку тогда и только тогда, когда эту строку можно прочитать по пути из корня до какой-то отмеченной терминальной вершины.

Хранение

Бор должен поддерживать три основные операции: вставку, удаление и поиск (определение существования). Пусть nnn – длина строки, над которой производится одна из этих операций.

Есть четыре основных способа хранить бор, они все завязаны на способах навигации в вершинах бора. Пусть вершина xxx имеет kkk детей x1,x2,…,xkx_1, x_2, \dots, x_kx1​,x2​,…,xk​, а ребра до детей имеют символы a1,a2,…,aka_1, a_2, \dots, a_ka1​,a2​,…,ak​.

Тогда навигация в вершине xxx осуществляется через

  1. Динамический массив пар (ai,xi)(a_i, x_i)(ai​,xi​). Все операции выполняются за время O(σn)O(\sigma n)O(σn).

  2. Статический массив вершин. Длина такого массива будет равна σ\sigmaσ. По индексу, соответствующему символу aia_iai​, будет храниться вершина xix_ixi​, а по остальным индексам – нейтральные значения. Все операции выполняются за время O(n)O(n)O(n).

  3. Сбалансированное BST, построенное на парах (ai,xi)(a_i, x_i)(ai​,xi​), где aia_iai​ играет роль ключа. Все операции будут выполняться за время O(nlog⁡σ)O(n \log \sigma)O(nlogσ).

  4. Хеш-таблица с ключами aia_iai​ и значениями xix_ixi​. Все операции выполняются за время O(n)O(n)O(n).

В зависимости от выбранного способа реализация самой структуры различается. Будем считать, что дети текущей вершины хранятся как ассоциативный массив, в котором ключом является символ.

struct vertex: dict[char: vertex] children bool is_terminal some additional data

На каждую вершину приходится O(σ)O(\sigma)O(σ) памяти, значит бор занимает память O(σm)O(\sigma m)O(σm), где mmm – суммарная длина всех строк, хранящихся в боре. У строк может не быть пересечений по префиксам.

Поиск

Обход бора совершается из его корня по ребрам, содержащими символы ключа, пока возможно. Если пройдя по всем символам ключа мы попали в терминальную вершину, то это значит, что ключ есть в боре, и ему соответствует найденная вершина. Если конечная вершина терминальная или в какой-то момент нужное ребро отсутствует, значит ключа в этом боре нет.

Поиск занимает время O(n)O(n)O(n), где nnn – длина искомого ключа.

function find(vertex root, string key) -> vertex: i = 0 while root != none and i < length of key: root = root.children[key[i]] i++ if root == none or root.is_terminal: return root else: return none

Вставка

function insert(vertex root, string key): for i = 0; i < length of key: if root.children[key[i]] == none: root.children[key[i]] = vertex() root = root.children[key[i]] root.is_terminal = true
