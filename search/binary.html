Бинарный поиск

Бинарный поиск – алгоритм быстрого нахождения элемента в отсортированных последовательностях. Выполняется за время O(log⁡n)O(\log n)O(logn).

Алгоритм бинарного поиска.

  1. Установить диапазон поиска от начала до конца последовательности.

  2. Сравнить средний элемент диапазона с искомым. Если он больше, то передвинуть правую границу поиска на середину. Если меньше, то передвинуть левую границу на середину.

  3. Повторять, пока не найдем элемент или пока в диапазоне поиска останется один элемент.

function binary_search(sequence s, target): int l = 0, r = length of s - 1 while l <= r: int m = (l + r) / 2 if s[m] == target: return m else if s[m] < target: l = m + 1 else: r = m - 1 return none

Вещественный бинарный поиск

Пусть есть некоторый бинарный критерий, для действительных чисел. Все числа, для которых этот критерий выполняется, меньше тех, для которых этот критерий не выполняется. То есть до какой-то границы критерий выполняется, а после нее – нет. Нам нужно найти эту границу.

Возьмем два числа lll и rrr. Для числа lll этот критерий гарантированно выполняется, а для числа rrr гарантированно не выполняется. Теперь мы можем воспользоваться идеей бинарного поиска. Будем уменьшать границу поиска, поддерживая этот инвариант: для lll критерий всегда выполняется, а для rrr всегда не выполняется.

Вещественный бинарный поиск

  1. Выбрать два числа lll и rrr – диапазон поиска.

  2. Посчитать среднее арифметическое m= racl+r2m = \ rac{l+r}{2}m= racl+r2, проверяем для него выполнение критерия.

  3. Если критерий выполняется, то присвоить новую границу l=ml = ml=m.
    Если не выполняется, то присвоить r=mr = mr=m.

  4. Продолжаем сужать диапазон до достижения точности ε\epsilonε.

Так мы нашли диапазон (l,r)(l, r)(l,r) с точностью ε\epsilonε, внутри которого находится граница выполнения критерия.

function binary_search(real l, real r): while r - l > ε: real m = (l + r) / 2 if criterion(m): l = m else: r = m return l, r
