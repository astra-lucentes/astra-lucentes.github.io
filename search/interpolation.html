Интерполяционный поиск

Давайте попробуем найти слово «абака» в словаре. Никто не будет искать его в середине, ведь оно начинается на А и точно находится на первых страницах. Этот подход, в отличие от бинарного поиска, учитывает возможное положение слова в словаре.

Используя эту идею, будем находить элемент xxx в массиве aaa. Ищем подобно бинарному поиску, но новый индекс вычисляем на основе расстояния между xxx и значениями на концах диапазона. Если границы lll и rrr, элементы на концах равны соответственно ala_lal​ и ara_rar​, то индекс mmm вычисляется из уравнения

x−alm−l=ar−alr−l ⟺ m=l+x−alar−al⋅(r−l)\frac{x-a_l}{m-l} = \frac{a_r-a_l}{r-l} ~\Longleftrightarrow~ m = l + \frac{x-a_l}{a_r-a_l} \cdot (r - l)m−lx−al​​=r−lar​−al​​ ⟺ m=l+ar​−al​x−al​​⋅(r−l)

Другими словами, бинарный поиск учитывает лишь знак разности между ключом и текущим значением, а интерполяционный ещё учитывает и модуль этой разности и по данному значению производит предсказание позиции следующего элемента для проверки.

function interpolation_search(array[int] a, int x): int l = 0, r = length of a - 1 while a[l] < x and x < a[r]: m = l + (x - a[l]) / (a[r] - a[l]) * (r - l) if a[m] < x: l = m + 1 else if a[m] > x: r = m - 1 else: return m if a[l] == x: return l else if a[r] == x: return r else: return none
