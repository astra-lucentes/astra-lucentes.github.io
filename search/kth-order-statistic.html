Поиск kkk-й порядковой статистики

kkk-я порядковая статистика коллекции – это kkk-й по величине элемент, то есть находящийся на kkk-м месте в отсортированном наборе элементов коллекции.

Если kkk-ю порядковую статистику нужно находить многократно, то целесообразней отсортировать массив за время O(nlog⁡n)O(n \log n)O(nlogn) и находить kkk-й элемент в отсортированном массиве за O(1)O(1)O(1).

Для qqq запросов общее время составит O(nlog⁡n+q)O(n \log n + q)O(nlogn+q).

Но для одноразового нахождения kkk-й порядковой статистики такой подход слишком неэффективен.

Алгоритм быстрого выбора

Идея алгоритма такая же, как и в быстрой сортировке. Время работы недетерминированное линейное.

После разделения массива на две части относительно опорного элементы мы можем определить позицию этого опорного элемента в отсортированном массиве.

Алгоритм быстрого выбора.

  1. Выбрать опорный элемент – случайный элемент.

  2. Разделить массив на две части: элементы, меньшие опорного и элементы, большие опорного.

  3. Найти mmm – количество элементов в первой части, то есть место опорного элемента.

  4. Если k=mk = mk=m, то мы нашли kkk-ю порядковую статистику – опорный элемент.
    Если k<mk < mk<m, то мы продолжаем искать kkk-ю порядковую статистику в первой части массива.
    Если k>mk > mk>m, то мы ищем (k−m−1)(k-m-1)(k−m−1)-ю порядковую статистику во второй части массива.

function order_statistic(array a, int k): l, r = 0, length of a forever: pivot = random(l, r-1) m = 0 for i = l; i < r and i != pivot: if a[i] <= a[pivot]: m++ if m == k: return a[m] else if k < m: r = m else: l = m

Алгоритм BFPRT.

Наилучшая временная сложность в прошлом алгоритме достигается, если в качестве опорного элемента выбирается медиана массива. Для точного нахождения медианы, придется или отсортировать массив, или найти n2\frac n22n​-ю порядковую статистику, или еще что-нибудь сложное.

Вместо выбора медианы можно выбрать элемент, близкий к медиане, и на его основе разделить массив.

Алгоритм медиана медиан. Для алгоритма нужно определить нечетное число s>3s > 3s>3 – размер группы..

  1. Если длина массива не больше sss, то отсортировать его, найти медиану и вернуть ее.

  2. Массив делится на группы по sss элементов в каждой. Остаток игнорируется.

  3. Для каждой группы ищется медиана. Получаем массив из ⌈ns⌉\b\lceil\frac ns \b\rceil⌈sn​⌉ элементов.

  4. Продолжаем этим же алгоритмом искать медиану полученного массива.
