Тернарный поиск

Тернарный (троичный) поиск – поиск, применяемый для нахождения максимума или минимума унимодальной функции или последовательности. Выполняется за время O(log⁡n)O(\log n)O(logn).

Поиск максимума последовательности

Алгоритм троичного поиска максимума последовательности.

  1. Разбить диапазон поиска на 3 части, сравнить значения на границах этих частей.

  2. Если значения возрастают, изменить левую границу диапазона на меньшую границу разбиения.

  3. Если значения убывают, изменить правую границу диапазона на большую границу разбиения.

  4. Продолжать, пока не останется 3 элемента. Затем возвращаем максимум из них.

function ternary_search_max(sequence s): left, right = 0, length of s while left < right - 3: left_third = (2 * left + right) / 3 right_third = (left + 2 * right) / 3 if s[left_third] < s[right_third]: left = left_third else: right = right_third return max(s[left], s[left+1], s[left+2])

Для поиска минимума последовательности достаточно заменить условие на обратное.

Поиск максимума функции

Алгоритм троичного поиска точки максимума функции.

  1. Разбить диапазон поиска на 3 части, сравнить значения на границах этих частей.

  2. Если значения возрастают, изменить левую границу диапазона на меньшую границу разбиения.

  3. Если значения убывают, изменить правую границу диапазона на большую границу разбиения.

  4. Продолжать, пока не достигнем желаемой точности. Максимум функции всегда находится между границ разбиения.

function ternary_search_argmax(f, real l, real r): while r - l >= ε: x1 = l + (r - l) / 3 x2 = r - (r - l) / 3 if f(x1) < f(x2): l = x1 else: r = x2 return (l + r) / 2

На каждой итерации мы дважды вычисляем значение функции и сокращаем диапазон поиска в полтора раза. Время работы алгоритма составляет 2log⁡32(r−lε)=Θ(log⁡1ε)2 \log_\frac32 \bl( \frac{r-l}{\epsilon} \br) = \Theta \b( \log \frac 1 \epsilon \b)2log23​​(εr−l​)=Θ(logε1​).

Вместо достижения точности ε\epsilonε можно просто выполнять фиксированное число итераций.

Метод золотого сечения

Метод золотого сечения применим для нахождения максимума или минимума унимодальной функции на отрезке. Он позволяет уменьшить количество вычислений функции за итерацию с 222 до 111 раза.

Идея состоит в делении отрезка на 333 части так, чтобы на следующей итерации одна из точек нового разбиения совпала с одной из точек текущего разбиения.

Здесь мы разделили отрезок [l,r][l, r][l,r] точками x1x_1x1​ и x2x_2x2​ на три части с длинами aaa, bbb и ccc.

Для того, чтобы точка x1x_1x1​ совпала с точкой следующего деления y2y_2y2​, необходимо выполнение соотношений

a+bc=ab=cb=φ\frac{a+b}{c} = \frac{a}{b} = \frac{c}{b} = \phica+b​=ba​=bc​=φ

Отсюда получаем, что a+b=φca+b = \phi ca+b=φc, а a=c=φba = c = \phi ba=c=φb. Значит, φ+1=φ2\phi + 1 = \phi^2φ+1=φ2 и φ=1+52\phi = \frac{1 + \sqrt5}{2}φ=21+5​​ – золотое сечение.

Алгоритм остается тем же, что и в обычном тернарном поиске. Изменяется только способ вычисления x1 и x2, а так же добавляется сохранение уже вычисленных значений функции.

function golden_search_argmax(f, real l, real r): x1 = l + (2 - φ) * (r - l) x2 = r - (2 - φ) * (r - l) f1, f2 = f(x1), f(x2) while r - l >= ε: if f1 < f2: l, x1 = x1, x2 f1 = f2 x2 = r - (2 - φ) * (r - l) f2 = f(x2) else: r, x2 = x2, x1 f2 = f1 x1 = l + (2 - φ) * (r - l) f1 = f(x1) return (l + r) / 2

На каждой итерации мы один раз вычисляем значение функции и сокращаем диапазон поиска в φ\phiφ раз. Время работы алгоритма составляет log⁡φ(r−lε)=Θ(log⁡1ε)\log_\phi \bl( \frac{r-l}{\epsilon} \br) = \Theta \b( \log \frac 1 \epsilon \b)logφ​(εr−l​)=Θ(logε1​).

Метод Фибоначчи

Идею разбивать отрезок поиска так, чтобы какие-то точки текущей и следующей итераций совпали, можно применить и для последовательностей.

Метод Фибоначчи позволяет находить максимум или минимум унимодальной последовательности. На каждой итерации мы получаем значение только одного элемента последовательности, вместо двух в обычном тернарном поиске.

Аналогом золотого сечения здесь служат числа Фибоначчи. Если длина отрезка поиска равна FnF_nFn​, то его следует разбить на отрезки Fn−2+Fn−3+Fn−2F_{n-2} + F_{n-3} + F_{n-2}Fn−2​+Fn−3​+Fn−2​.

Что делать, если длина последовательности не является числом Фибоначчи? Взять первое число Фибоначчи, которое не меньше длины последовательности, и первой операцией отрезать лишний хвост. Для этого придется в начале дополнительно дважды получить элемент последовательности.
